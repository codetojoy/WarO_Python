
### Summary

* We assume that pipenv is already installed:
    - `brew install pipenv`
* To update `Pipfile.lock`:
    - `pipenv install`
* To run virtual environment:
    - `pipenv shell`
    - Presumably, in this env, pytest is already installed. (I'm not sure how
to test this exactly.)
